**Step 6: Add sendAnswer to WebSocketClient**

```kotlin
suspend fun sendAnswer(
    kuerzel: String,
    type: String,          // "single", "multi", or "text"
    selections: List<Int>, // selected option numbers (1-based)
    text: String?,         // free text for "text" type
    optionCount: Int       // total number of options
) {
    val payload = buildMap<String, Any> {
        put("action", "answer")
        put("kuerzel", kuerzel)
        put("type", type)
        put("selections", selections)
        put("option_count", optionCount)
        if (text != null) put("text", text)
    }
    send(json.encodeToString(payload))
}
```

Note: kotlinx.serialization cannot directly encode `Map<String, Any>`. Instead, use `buildJsonObject` from `kotlinx.serialization.json`:
```kotlin
import kotlinx.serialization.json.buildJsonObject
import kotlinx.serialization.json.put
import kotlinx.serialization.json.putJsonArray
import kotlinx.serialization.json.JsonPrimitive

suspend fun sendAnswer(kuerzel: String, type: String, selections: List<Int>, text: String?, optionCount: Int) {
    val payload = buildJsonObject {
        put("action", "answer")
        put("kuerzel", kuerzel)
        put("type", type)
        putJsonArray("selections") { selections.forEach { add(JsonPrimitive(it)) } }
        put("option_count", optionCount)
        if (text != null) put("text", text)
    }
    send(payload.toString())
}
```

**Step 1: Overhaul QuestionCard.kt**

Replace the current QuestionCard that parses options from message text with a new implementation that uses `ChatMessage.questionData`.

The new QuestionCard must handle three modes:

**Single-choice mode** (`questionData.multiSelect == false`, has options):
- Show question text from `questionData.question` (with header prefix if present)
- Render each option as a tappable chip/button showing `"{index}. {label}"` and description if present
- When user taps an option, call `onAnswer` with type="single", selections=[index] (1-based)
- After answered, highlight the selected option, disable all others

**Multi-choice mode** (`questionData.multiSelect == true`):
- Same question display
- Render options as toggleable FilterChips (selected state tracked locally via `remember { mutableStateListOf<Int>() }`)
- Add a "Submit" button below the options (enabled only when at least 1 option is selected)
- When user taps Submit, call `onAnswer` with type="multi", selections=[1,3,...] (1-based indices of selected options)
- After answered, show selected options highlighted

**Free-text mode** (no options OR user selects "Other"):
- For questions with options: add an "Other..." chip/button at the end
- When "Other" is tapped, show a TextField below the options
- Add a "Send" button next to the TextField
- When user taps Send, call `onAnswer` with type="text", text=fieldValue
- For questions with no options: show only the TextField + Send button
