<objective>
Add WebSocket protocol support for session creation: new DTO types, parser logic, domain models, send methods, and repository interface for `list_directories` and `create_session` actions.

Purpose: The app currently cannot send or receive session-creation protocol messages. The RelayMessageTypeDto enum is missing `directory_list` and `session_created` variants, so server responses are silently dropped. This plan adds the entire shared-module protocol layer that the UI (Plan 02) depends on.

Output: Shared module can send `list_directories`/`create_session` actions and parse their responses into domain objects.
</objective>
