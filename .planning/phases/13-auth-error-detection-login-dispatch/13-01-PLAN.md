Add auth error detection and recovery state management to relay-server.cjs. All additions go in the server file — no new files.

**1. Auth recovery state (add after line ~613, near lastStatusMap):**

```javascript
// --- Auth Recovery State ---
const authRecoveryState = new Map(); // kuerzel -> { state: 'detected'|'login_sent'|'waiting_url'|'recovered', detectedAt: number }
const AUTH_SCAN_COOLDOWN = 30000; // 30s per session
const AUTH_RECOVERY_TIMEOUT = 300000; // 5 minutes
const lastAuthScanMap = new Map(); // kuerzel -> timestamp of last scan

const AUTH_ERROR_PATTERN = /(session has expired|oauth token (has expired|revoked)|not logged in|please run \/login|API Error: 401|authentication_error)/i;
```

**2. Auth scan function (add as new function near the status polling section):**

Create `async function scanForAuthError(kuerzel)`:
- Check cooldown: if `lastAuthScanMap.get(kuerzel)` is within 30s, return
- Check recovery state: if `authRecoveryState.has(kuerzel)`, skip (already in recovery)
- Update `lastAuthScanMap.set(kuerzel, Date.now())`
- Use existing `getLastResponses(kuerzel, 1)` to get the last screen response
- If response is null or empty, return
- Test the response text against `AUTH_ERROR_PATTERN`
- If match found:
  - Set `authRecoveryState.set(kuerzel, { state: 'detected', detectedAt: Date.now() })`
  - Log: `[relay-server] Auth error detected for @${kuerzel}: ${match[0]}`
  - Send `auth_required` message to app: `appSocket.send(JSON.stringify({ type: 'auth_required', session: kuerzel, error: match[0], timestamp: Date.now(), __relay: true }))`
  - Call `dispatchCommand(kuerzel, '/login')`
  - Update state: `authRecoveryState.get(kuerzel).state = 'login_sent'`
  - Log: `[relay-server] Dispatched /login to @${kuerzel}`

**3. Hook into startStatusPolling (modify the status change detection block around line 662):**

Inside the `if (lastStatusMap.get(kuerzel) !== status)` block, AFTER updating lastStatusMap and sending the status message, add:

```javascript
// Auth error detection: scan when session transitions to ready (may have stopped due to auth error)
const previousStatus = lastStatusMap.get(kuerzel); // Note: must capture BEFORE the set() call above
```

Adjust the code so `previousStatus` is captured before `lastStatusMap.set(kuerzel, status)`. Then after the status broadcast:

```javascript
if (previousStatus === 'working' && status === 'ready') {
  scanForAuthError(kuerzel);
}
```

**4. Recovery timeout check (add inside the poll function, before the pane iteration):**

```javascript
// Check for auth recovery timeouts
for (const [kuerzel, recovery] of authRecoveryState) {
  if (Date.now() - recovery.detectedAt > AUTH_RECOVERY_TIMEOUT) {
    process.stderr.write(`[relay-server] Auth recovery timeout for @${kuerzel}, resetting\n`);
    authRecoveryState.delete(kuerzel);
  }
}
```

**5. Recovery success detection (add in the status change block):**

When a session that is in `authRecoveryState` transitions to 'working', it means recovery succeeded:

```javascript
if (authRecoveryState.has(kuerzel) && status === 'working') {
  process.stderr.write(`[relay-server] Auth recovery successful for @${kuerzel}\n`);
  authRecoveryState.delete(kuerzel);
}
```

**6. Clear auth state on disconnect (in stopStatusPolling):**

Add `authRecoveryState.clear(); lastAuthScanMap.clear();` to `stopStatusPolling()`.

**Important:** The `scanForAuthError` function must guard against `appSocket` being null or not open (readyState !== 1) before sending — same pattern used in the polling loop.
