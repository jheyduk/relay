Add OAuth URL scanning to the relay-server polling loop. Implements AUTH-03 and server-side AUTH-04.

1. Add a new constant `OAUTH_URL_PATTERN` right after `AUTH_ERROR_PATTERN` (line ~621):
   ```javascript
   const OAUTH_URL_PATTERN = /(https:\/\/(?:console\.anthropic\.com|claude\.ai)\/oauth\S+)/i;
   ```

2. Add a new `URL_SCAN_INTERVAL` constant (3 seconds):
   ```javascript
   const URL_SCAN_INTERVAL = 3000;
   ```
   Also add a `lastUrlScanMap` (kuerzel -> timestamp) parallel to `lastAuthScanMap`.

3. Create a new async function `scanForOAuthUrl(kuerzel)` after `scanForAuthError()`:
   - Guard: only proceed if `authRecoveryState.get(kuerzel)?.state === 'login_sent'`
   - Guard: cooldown check against `lastUrlScanMap` with `URL_SCAN_INTERVAL`
   - Update `lastUrlScanMap.set(kuerzel, Date.now())`
   - Call `getLastResponses(kuerzel, 1)` — this returns a **string** (NOT an array, do NOT call .map() on it)
   - Test `OAUTH_URL_PATTERN.exec(responseText)` against the string
   - If match found:
     - Update recovery state: `recovery.state = 'url_sent'`
     - Send to app: `appSocket.send(JSON.stringify({ type: 'auth_url', session: kuerzel, url: match[1], timestamp: Date.now(), __relay: true }))`
     - Log: `process.stderr.write('[relay-server] OAuth URL extracted for @${kuerzel}: ${match[1]}\n')`
   - If no match: do nothing (will retry next poll cycle)

4. In the `poll()` function inside `startStatusPolling()`, add URL scanning after the auth timeout check block (after line ~713, before the status change loop):
   ```javascript
   // Scan for OAuth URL in sessions that are in login_sent state
   for (const [kuerzel, recovery] of authRecoveryState) {
     if (recovery.state === 'login_sent') {
       scanForOAuthUrl(kuerzel);
     }
   }
   ```

5. Modify the existing auth recovery timeout block (line ~708-713) to send `auth_timeout` to the app before deleting:
   ```javascript
   if (Date.now() - recovery.detectedAt > AUTH_RECOVERY_TIMEOUT) {
     process.stderr.write(`[relay-server] Auth recovery timeout for @${kuerzel}, resetting\n`);
     if (appSocket && appSocket.readyState === 1) {
       appSocket.send(JSON.stringify({
         type: 'auth_timeout',
         session: kuerzel,
         timestamp: Date.now(),
         __relay: true,
       }));
     }
     authRecoveryState.delete(kuerzel);
   }
   ```

6. In `stopStatusPolling()`, also clear `lastUrlScanMap`:
   ```javascript
   lastUrlScanMap.clear();
   ```

IMPORTANT: `getLastResponses(kuerzel, count)` returns a string. Do NOT call `.map()`, `.forEach()`, or treat it as an array. This was a server-crashing bug fixed in Phase 13.
