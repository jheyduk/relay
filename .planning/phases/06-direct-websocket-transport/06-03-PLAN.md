    1. **Create NsdDiscovery.kt** at `androidApp/src/main/java/dev/heyduk/relay/service/NsdDiscovery.kt`:

    ```kotlin
    package dev.heyduk.relay.service

    import android.content.Context
    import android.net.nsd.NsdManager
    import android.net.nsd.NsdServiceInfo
    import kotlinx.coroutines.channels.awaitClose
    import kotlinx.coroutines.flow.Flow
    import kotlinx.coroutines.flow.callbackFlow
    import timber.log.Timber
    ```

    Class `NsdDiscovery(private val context: Context)`:
    - `private val nsdManager = context.getSystemService(Context.NSD_SERVICE) as NsdManager`
    - Method `discover(): Flow<NsdServiceInfo>` using `callbackFlow`:
      - Create `NsdManager.DiscoveryListener`:
        - `onServiceFound`: if `info.serviceType` contains `_relay._tcp`, call `nsdManager.resolveService(info, resolveListener)` where resolveListener's `onServiceResolved` does `trySend(resolved)`
        - `onServiceLost`: log
        - `onDiscoveryStarted`: log "mDNS discovery started"
        - `onDiscoveryStopped`: log
        - `onStartDiscoveryFailed`: log + `close()`
        - `onStopDiscoveryFailed`: log
      - Call `nsdManager.discoverServices("_relay._tcp.", NsdManager.PROTOCOL_DNS_SD, listener)`
      - `awaitClose { try { nsdManager.stopServiceDiscovery(listener) } catch (e: Exception) { Timber.w(e, "Stop discovery failed") } }`
    - Method `resolveToUrl(info: NsdServiceInfo): String`:
      - Return `"ws://${info.host.hostAddress}:${info.port}"`

    2. **Create WebSocketService.kt** at `androidApp/src/main/java/dev/heyduk/relay/service/WebSocketService.kt`:

    Same structure as PollingService but WebSocket-based. Key differences:
    - Reads `server_secret` and `wireguard_ip` from DataStore (not bot tokens)
    - Uses NsdDiscovery to find server, falls back to configured WireGuard IP

    ```kotlin

    import android.app.NotificationChannel
    ```

    Class `WebSocketService : Service()`:
    - Companion: `const val NOTIFICATION_ID = 1; const val CHANNEL_ID = "relay_connection"`
    - Inject: `dataStore`, `networkMonitor`, `webSocketClient: WebSocketClient`, `database: RelayDatabase`, `notificationHelper: NotificationHelper`, `nsdDiscovery: NsdDiscovery`
    - `serviceScope = CoroutineScope(SupervisorJob() + Dispatchers.IO)`
    - `@Volatile private var connectionStarted = false`

    `onStartCommand`:
    - Create notification channel (name: "Relay Connection", IMPORTANCE_LOW)
    - `startForeground(NOTIFICATION_ID, buildNotification("Connecting..."), FOREGROUND_SERVICE_TYPE_DATA_SYNC)`
    - Guard against multiple starts with `connectionStarted` flag
    - Launch in serviceScope:

      a. Read config from DataStore:
      ```kotlin
      val prefs = dataStore.data.first()
      val secret = prefs[stringPreferencesKey("server_secret")] ?: ""
      val wireguardIp = prefs[stringPreferencesKey("wireguard_ip")] ?: ""
      ```
      If secret is blank, log error, updateNotification("Missing configuration"), stopSelf(), return.

      b. Discover server URL — parallel mDNS + WireGuard:
      ```kotlin
      val serverUrl = discoverServer(wireguardIp)
      ```

      c. Collect updates and insert into database (same pattern as PollingService):
      ```kotlin
      launch {
          webSocketClient.updates.collect { update ->
              database.messagesQueries.insertOrIgnore(...)
              when (update.type) {
                  RelayMessageType.PERMISSION -> notificationHelper.showPermissionNotification(update)
                  RelayMessageType.COMPLETION -> notificationHelper.showCompletionNotification(update)
                  else -> {}
              }
          }
      }
      ```

      d. Update notification based on connection state:
      ```kotlin
      launch {
          webSocketClient.connectionState.collect { state ->
              val text = when (state) {
                  ConnectionState.CONNECTED -> "Relay connected"
                  ConnectionState.CONNECTING -> "Connecting..."
                  ConnectionState.DISCONNECTED -> "Disconnected — reconnecting"
              }
              updateNotification(text)
          }
      }
      ```

      e. Connect with network awareness:
      ```kotlin
      while (isActive) {
          networkMonitor.awaitConnected()
          try {
              webSocketClient.connectWithRetry(serverUrl, secret)
          } catch (e: CancellationException) { throw e }
          catch (e: Exception) { Timber.e(e, "WebSocket error, awaiting reconnect") }
      }
      ```

    Private `suspend fun discoverServer(wireguardIp: String): String`:
    - Start mDNS discovery and WireGuard attempt in parallel
    - mDNS: `nsdDiscovery.discover().firstOrNull()` with 5s timeout (`withTimeoutOrNull(5000)`)
    - If mDNS finds service: return `nsdDiscovery.resolveToUrl(info)`
    - Else if wireguardIp is not blank: return `"ws://$wireguardIp:9784"`
    - Else: throw Exception("No server found — configure WireGuard IP or ensure local network")

    Same `onTimeout`, `onDestroy`, notification helpers as PollingService. Channel ID changes to `"relay_connection"`.
