    1. Create `SessionRepository.kt` interface in `data/repository/`:
    ```kotlin
    package dev.heyduk.relay.data.repository
    import dev.heyduk.relay.domain.model.Session
    import kotlinx.coroutines.flow.Flow

    interface SessionRepository {
        /** Observable list of discovered sessions. Updated when /ls responses are parsed. */
        val sessions: Flow<List<Session>>

        /** Currently selected session kuerzel. */
        val selectedKuerzel: Flow<String?>

        /** Trigger session discovery by sending /ls command. */
        suspend fun refreshSessions()

        /** Select a session by kuerzel for command targeting. */
        fun selectSession(kuerzel: String?)

        /** Get the last response text for a specific session (from /last output or DB). */
        suspend fun getLastResponse(kuerzel: String): String?
    }
    ```

    2. Create `SessionRepositoryImpl.kt`:
    - Constructor takes `TelegramRepository` as dependency
    - `_sessions` is a `MutableStateFlow<List<Session>>(emptyList())`
    - `_selectedKuerzel` is a `MutableStateFlow<String?>(null)`
    - `sessions` and `selectedKuerzel` expose the StateFlows as `Flow`
    - `refreshSessions()`: calls `telegramRepository.sendRawCommand("/ls")`. The /ls response will arrive as a RelayUpdate via the polling loop. To capture it, collect `telegramRepository.updates` in the init block and filter for messages that look like /ls responses (contain lines matching `@\S+\s+\w+` pattern). When such a message arrives, parse with `SessionListParser.parse()` and update `_sessions`.
    - Implementation detail for /ls response detection: In the init block, launch a coroutine that collects `telegramRepository.updates`. For each update, try `SessionListParser.parse(update.message)`. If the result is non-empty (at least 1 session parsed), update `_sessions.value`. This is simple and avoids tracking request/response correlation.
    - `selectSession(kuerzel)`: sets `_selectedKuerzel.value = kuerzel`
    - `getLastResponse(kuerzel)`: collects first emission from `telegramRepository.getMessagesForSession(kuerzel)`, returns the last message's text, or null if empty

    The CoroutineScope for the init-block collection should be passed as a constructor parameter (default `CoroutineScope(Dispatchers.Default)`) following the same pattern as TelegramRepositoryImpl.
