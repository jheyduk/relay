  <action>
Create `server/install.cjs` — a standalone CommonJS script (NOT ESM like zellij-claude's install.js) that:

1. Reads `~/.claude/settings.json`
2. REMOVES any existing hook entries that contain `zellij-claude` in their command path (this cleans up the old hooks)
3. ADDS new hook entries pointing to the relay server hooks

The hook registration structure follows Claude Code's settings.json format (same as zellij-claude's install.js, but with new paths):

```
SessionStart: node "{__dirname}/hooks/session-start.cjs" (sync, not async — it must finish before Claude starts)
Notification(permission_prompt): node "{__dirname}/hooks/permission-notify.cjs" (async)
Stop: node "{__dirname}/hooks/session-stop.cjs" (async)
PreToolUse(AskUserQuestion): node "{__dirname}/hooks/ask-notify.cjs" (async)
```

Note: Do NOT register pretool-cache.cjs — that stays in zellij-claude (it caches tool input to /tmp for permission-notify to read, but it's a zellij-claude concern, not relay's).

The script must:
- Use CommonJS (`require()`) not ESM — consistent with all other server/ files
- Use `path.join(__dirname, 'hooks', 'session-start.cjs')` for paths (absolute paths in settings.json)
- When removing old zellij-claude hooks: filter out any hook entry whose command includes `zellij-claude` AND includes one of our hook names (session-start, session-stop, permission-notify, ask-notify). Do NOT remove the pretool-cache hook — that stays.
- Be idempotent — running twice should not create duplicate entries
- Print what was added and what was removed for user feedback
- Handle the case where settings.json doesn't exist yet (create it with just the hooks)

Run as: `node server/install.cjs` from anywhere (uses __dirname for absolute paths)

Also add an "install-hooks" script to server/package.json (read the file first, then edit to add the script without overwriting existing content):
```json
"scripts": {
  "start": "node relay-server.cjs",
  "install-hooks": "node install.cjs"
}
```
